# ⚡ 即興カスタム チームメイカー

LoL / VALORANT イベントの「即興カスタム」用ツール。
**参加者はQRコードを読み取ってエントリーするだけ**で、

1. 🎯 **ランク & 希望ポジションで自動チーム分け**（戦力バランス最適化）
2. 🏆 **トーナメント表（シングルエリミネーション）を自動生成**

までを一気に行えます。Python 標準ライブラリのみで動くので**追加インストール不要**です。

---

## 🚀 使い方（クイックスタート）

### 1. 起動する
`start.bat` をダブルクリック（または下記コマンド）。

```powershell
python server.py
```

自動でブラウザの **主催者ダッシュボード** が開きます。
（開かない場合は `http://localhost:8080/admin` にアクセス）

### 2. イベントを作成
- ゲーム（LoL / VALORANT）を選択
- イベント名・1チームの人数を入力 → **「イベントを作成」**

### 3. 参加者にQRを見せる
- **STEP1 エントリー受付** に表示されるQRコードを参加者にスクリーン／プロジェクターで提示
- 参加者がスマホで読み取り → 名前・ランク・希望ポジションを入力して送信
- エントリーは**ダッシュボードにリアルタイム反映**されます

#### 🌐 オンライン公開（同じWi-Fi不要 / おすすめ）
本ツールは **Cloudflare Tunnel**（`cloudflared.exe` 同梱・アカウント登録不要・無料）で、
ローカルサーバーを一時的にインターネット公開します。

- 起動後 数秒〜十数秒で QRコード横の表示が **「🌐 オンライン公開中」**（緑）に変わります
- この状態のQRは **4G/5G・別のWi-Fi など、どの回線からでも** 読み取り＆エントリー可能
- QR上部の **「🌐 オンライン / 📶 同じWi-Fi」** ボタンでいつでも切替できます

> 💡 公開URL（`https://◯◯◯.trycloudflare.com`）は **起動するたびに変わります**。
> 毎回その場でQRを表示するだけなので運用上は問題ありません。
> オンライン公開を使わない場合は「📶 同じWi-Fi」を選べばLAN内のみで動きます
> （その場合は参加者と主催PCが同じWi-Fiに接続している必要があります）。

> ⚙️ オンライン公開を無効化したいとき: 環境変数 `NO_TUNNEL=1` をセットして起動。

#### 🔗 常設URL（毎回同じ固定URL / 独自ドメイン）
通常のオンライン公開URLは起動のたびに変わりますが、**お持ちの独自ドメイン**を使えば
`https://customs.yourteam.gg` のような **固定URL** にできます（無料・一度だけ設定）。

**前提**: ドメインが Cloudflare に登録済みであること
（ドメインのネームサーバーが Cloudflare を指している状態。未登録なら
[Cloudflareダッシュボード](https://dash.cloudflare.com) の「サイトを追加」から登録）。

**設定手順（一度だけ）**:
1. `setup-tunnel.bat` をダブルクリック
2. 画面の案内に従う:
   - ブラウザでCloudflareにログイン →使うドメインを選んで「Authorize」
   - 使いたいホスト名を入力（例 `customs.yourteam.gg`）
3. 「完了しました！」と出れば設定完了

以降は **`start.bat` を起動するだけ**で、毎回その固定URLで公開されます
（QR・エントリーURLも自動で固定URLになり、管理画面に「固定URLで公開中」と表示）。

- 設定は `tunnel.json` に保存されます
- **使い捨てURLに戻す**には `tunnel.json` を削除してください

> ⚠️ 固定URLでも「PCで `start.bat` を起動している間だけ」アクセス可能です。
> PCを閉じている間もずっとアクセスできる**24時間常設**にしたい場合は、
> 外部クラウドへのデプロイが必要です（別途ご相談ください）。

### 4. チーム分け
- **STEP2 チーム分け** → **「チーム分けを生成」**
- 戦力差が表示されます。気に入らなければ **「再シャッフル」**
- メンバーを **ドラッグ&ドロップ** で手動入れ替えも可能
- チーム名はクリックで編集可能

### 5. トーナメント
- **STEP3 トーナメント** → シード方式を選んで **「表を作成」**
- 各試合の **勝者をクリック** すると次の試合へ自動で勝ち上がり
- 優勝が決まると 👑 チャンピオン表示

---

## 🧠 チーム分けの仕組み

| 項目 | 内容 |
|------|------|
| ランク評価 | ティア + ディビジョンを内部スコア化（LoL: Iron〜Challenger / VALO: Iron〜Radiant） |
| バランス調整 | 強い順に「最も戦力の低いチーム」へ割り当て、合計戦力を均等化 |
| ポジション (LoL) | 各チームに TOP/JG/MID/ADC/SUP を1人ずつ配置。第1→第2希望→空き枠の順 |
| ポジション (VALO) | ロールは自由。希望ロールが偏らないよう分散も考慮 |
| 端数 | チーム数に満たない人は **補欠** として表示 |

割り当ての希望度合いは各メンバーに **第2希望 / オフロール** タグで表示されます。

---

## ⚙️ 設定・カスタマイズ

- **ポート変更**: 環境変数 `PORT` を設定（例 `set PORT=9000 && python server.py`）
- **ランクの重み付け**: `server.py` 上部の `RANKS` を編集
- **データ保存先**: 同フォルダの `data.json`（イベント・エントリーが保存されます）

## ❓ よくある質問

**Q. 参加者がQRを読んでもページが開かない**
A. QR横が **「🌐 オンライン公開中」**（緑）になっているか確認してください。なっていれば
どの回線からでも開けます。まだ準備中（黄）の場合は数秒待ってから再表示してください。
「📶 同じWi-Fi」モードの場合は、参加者と主催PCが同じWi-Fiに繋がっている必要があります。

**Q. オンライン公開が「準備中」のまま進まない / 失敗する**
A. 主催PCがインターネットに接続されているか確認してください。会社・学校などのネットワークで
`cloudflared` の通信がブロックされている場合は、主催PCをスマホのテザリングに繋ぐと確実です。
どうしても繋がらない場合は「📶 同じWi-Fi」モードでLAN運用に切り替えてください。

**Q. ネットが無い会場でも使える？**
A. はい。「📶 同じWi-Fi」モードならLAN内で完結し、インターネット接続は不要です
（QRライブラリも同梱済み）。オンライン公開機能だけはインターネットが必要です。

**Q. 途中でPCを閉じてもデータは残る？**
A. `data.json` に保存されるので、再起動後も同じイベントを継続できます。
